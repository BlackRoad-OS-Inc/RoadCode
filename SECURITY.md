# Security Policy

> Security is sovereignty. Your data, your hardware, your rules.

## Supported Versions

| Package | Version | Supported |
|---------|---------|-----------|
| @roadcode/greenlight | 0.1.x | Yes |
| @roadcode/greenlight-cli | 0.1.x | Yes |
| @roadcode/roadchain | 0.1.x | Yes |
| roadcode.blackroad.io (site) | latest | Yes |

## Reporting a Vulnerability

If you discover a security vulnerability in any BlackRoad OS project, **please report it responsibly**.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, email: **security@blackroad.io**

Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 1 week
- **Fix:** Depends on severity, but we prioritize security

## Our Commitment

- We will never blame reporters
- We will credit reporters (unless they prefer anonymity)
- We fix vulnerabilities — we don't hide them
- We default to transparency once a fix is available

## Scope

This policy covers:

- All packages in the `packages/` directory
- The static site at roadcode.blackroad.io
- GitHub Actions workflows
- Any infrastructure described in our specifications

## Philosophy

Security in BlackRoad OS follows the same values as everything else:

- **Consent** — We don't collect data you didn't agree to share
- **Care** — When something breaks, we help, we don't blame
- **Sovereignty** — Your hardware, your data, your rules
- **Equality** — Security isn't a premium feature
